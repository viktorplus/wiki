# Шпаргалка по команде `git clean`

Команда `git clean` используется для **удаления неотслеживаемых файлов и папок** в рабочем каталоге Git. Эти файлы не отслеживаются Git (например, временные файлы, сгенерированные системой сборки или IDE), но они могут занимать место в проекте.

**Опасность:** `git clean` удаляет файлы **безвозвратно**, и их нельзя будет восстановить через Git, если они не были добавлены в индекс.

---

## 1. Синтаксис

```bash
git clean [-d] [-f] [-x] [-n] [<path>...]
````

### Опции:

- `-f` — **force** (принудительно), необходимо для удаления. Без этой опции команда ничего не сделает.
    
- `-d` — удаляет также **неотслеживаемые директории**.
    
- `-x` — удаляет файлы, игнорируемые `.gitignore`. Без этой опции только неотслеживаемые файлы будут удаляться, а игнорируемые останутся.
    
- `-n` — **dry-run** (показать, что будет удалено, но не удалять). Полезно для проверки перед настоящим удалением.
    

---

## 2. Примеры использования

### 2.1. `git clean -n`

```bash
git clean -n
```

**Что делает:**

- Показывает, какие файлы будут удалены, но **не удаляет их**.
    
- Используется как **предупреждение**: перед настоящим удалением ты можешь увидеть, что именно будет убрано.
    

**Пример вывода:**

```text
Would remove file1.txt
Would remove file2.log
Would remove build/
```

---

### 2.2. `git clean -f`

```bash
git clean -f
```

**Что делает:**

- Удаляет все **неотслеживаемые файлы**, но оставляет **неотслеживаемые папки**.
    
- Если ты хочешь удалить файлы, но оставить каталоги, используй эту команду.
    

---

### 2.3. `git clean -fd`

```bash
git clean -fd
```

**Что делает:**

- Удаляет все **неотслеживаемые файлы** и **неотслеживаемые папки**.
    

Если у тебя в проекте есть временные папки вроде `build/`, `node_modules/`, `dist/` и они не отслеживаются Git, эта команда их удалит.

---

### 2.4. `git clean -fx`

```bash
git clean -fx
```

**Что делает:**

- Удаляет все неотслеживаемые файлы, включая те, что были проигнорированы через `.gitignore`.
    
- Это полезно, если ты хочешь очистить проект от всех временных файлов, включая те, что игнорируются Git.
    

**Пример:**

Если в `.gitignore` есть строки:

```text
*.log
*.tmp
```

`git clean -fx` удалит все файлы `.log`, `.tmp` в проекте, даже если они добавлены в `.gitignore`.

---

### 2.5. `git clean -fdx`

```bash
git clean -fdx
```

**Что делает:**

- Удаляет **все** неотслеживаемые файлы и директории, включая те, что игнорируются `.gitignore`.
    
- Используется для **очистки всего** рабочего каталога от временных файлов, даже тех, которые ты явно проигнорировал в `.gitignore`.
    

---

## 3. Важные замечания

1. **Будьте осторожны с `git clean -f`**:
    
    - Удаление файлов необратимо! Если ты случайно забудешь использовать `-n` (dry-run), файлы удалятся без возможности восстановления через Git.
        
2. **Не удаляет отслеживаемые файлы**:  
    Команда `git clean` **не затронет файлы**, которые уже находятся под версионным контролем Git (они были добавлены через `git add` и закоммичены).
    
3. **Не удаляет `.git` директорию**:  
    `.git` — это папка, которая содержит все метаданные Git, она никогда не будет удалена командой `git clean`.
    
4. **Рекомендуется использовать `-n` перед реальным удалением**:  
    Всегда проверяй, что будет удалено с помощью `git clean -n` перед тем, как выполнять настоящую очистку.
    

---

## 4. Часто задаваемые вопросы

### Вопрос 1: Почему мне нужно использовать `git clean`?

**Ответ:**  
`git clean` полезен, когда ты хочешь удалить **неотслеживаемые файлы** и **папки**, такие как временные файлы, сгенерированные IDE, сборщики, кэш и прочее. Это помогает держать проект в чистоте и освобождать место на диске.

### Вопрос 2: Можно ли восстановить файлы после `git clean`?

**Ответ:**  
Нет. После того как ты использовал `git clean -f` (или без флага `-n`), файлы **не восстанавливаются** через Git. Если ты случайно удалил что-то важное, это будет потеряно. Рекомендуется всегда использовать `git clean -n` перед выполнением команды.

---

## 5. Резюме

|Команда|Описание|
|---|---|
|`git clean -n`|Показать, что будет удалено, но не удалять файлы|
|`git clean -f`|Удалить все **неотслеживаемые файлы**, но оставить папки|
|`git clean -fd`|Удалить все **неотслеживаемые файлы и папки**|
|`git clean -fx`|Удалить все **неотслеживаемые файлы, включая игнорируемые**|
|`git clean -fdx`|Удалить **всё** — все файлы и папки, включая игнорируемые|
