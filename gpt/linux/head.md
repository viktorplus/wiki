# head — шпаргалка

Коротко: `head` показывает начало файла/потока, удобно чтобы оценить формат, заголовки или первые строки логов.
#head, #linux, #logs
---

## Синтаксис

```
head [OPTIONS] [FILE...]
head -n 20 file.txt       # первые 20 строк
head -c 100 file.txt      # первые 100 байт
command | head -n 5       # первые 5 строк stdout
```

Если `FILE` не указан, читается stdin.

## Основные опции

- `-n NUM` / `--lines=NUM` — сколько строк показать (по умолчанию 10)
- `-c NUM` / `--bytes=NUM` — показать первые байты
- `-q` / `--quiet` — не печатать заголовки файлов
- `-v` / `--verbose` — всегда печатать имя файла
- `-n -5` — показать все, кроме последних 5 строк ('-' разность)
- `head -n +5` — показать начиная с первой строки до 5 включительно (допустимо `head -n +5 file`)

## Примеры и паттерны

- Быстрый просмотр начала логов:
  ```
  head /var/log/syslog
  ```
- Проверка заголовков CSV/TSV:
  ```
  head -n 1 data.csv
  ```
- Первые строки нескольких файлов:
  ```
  head -n 5 *.conf
  ```
- Просмотр вывода команды:
  ```
  ls -lt | head -n 10      # десять самых новых файлов
  ```
- Использование с `tail`:
  ```
  tail -n +20 logfile | head -n 10   # строки 20-29
  ```

## Советы

- `head` полезен для проверки формата: `head -n 1` показывает separator row
- В скриптах `head -n 1` используется для получения первой строки конфигурации или версии
- Для интерфейсов, где `head` вырезает начало больших бинарных файлов, добавьте `file`/`strings` для понимания формата
- Вместо `cat file | head` лучше `head file`, чтобы не создавать дополнительный pipe
