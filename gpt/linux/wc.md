# wc — шпаргалка

Коротко: `wc` (word count) считает строки, слова, байты и символы в файлах или stdin; часто используется для базового анализа размеров текстов.
#wc, #linux, #cli
---

## Синтаксис

```
wc [OPTIONS] [FILE...]
cat file.txt | wc -l   # показать число строк через pipe
wc -w notes.md        # число слов
wc -c binary.dat      # размер в байтах
```

Если `FILE` не указан, `wc` читает из stdin.

## Основные опции

| Опция | Описание |
|-------|----------|
| `-l`  | только количество строк |
| `-w`  | только количество слов |
| `-c`  | количество байтов |
| `-m`  | количество символов (UTF-8/многобайт) |
| `-L`  | длина самой длинной строки |
| `-h`  | человекочитаемый вывод (GNU wc) |

Комбинируйте опции, чтобы получить нужные метрики (`wc -lwm file`). По умолчанию `wc` печатает `lines words bytes` для каждого файла и суммарно.

## Примеры

- Подсчитать строки в каталоге:
  ```
  find . -name '*.log' -print0 | xargs -0 wc -l
  ```
- Суммировать слова из нескольких файлов:
  ```
  wc -w README.md docs/*.md
  ```
- Проверить длину самой длинной строки:
  ```
  wc -L sql/schema.sql
  ```
- Использовать `wc` в pipe со `grep`:
  ```
  grep -R "TODO" src | wc -l
  ```
- Узнать размер файла (в байтах и символах):
  ```
  wc -cm data.csv
  ```

## Советы

- При подсчёте строк в бинарных файлах убедитесь, что `wc` не ломает вывод (используйте `file`/`strings`).
- `wc -l` часто используется в скриптах для проверки, вернул ли `grep` строки.
- Чтобы измерить размер stdin (например, `curl`), добавьте `| wc -c`.
- В `make` или CI удобно делать `if [ $(wc -l < file) -gt 100 ]; then ...; fi`.
