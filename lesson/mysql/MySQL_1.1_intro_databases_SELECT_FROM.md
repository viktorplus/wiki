# MySQL — Урок 1.1. Введение в базы данных (SELECT / FROM)

## 1) База данных и СУБД
**База данных (DB)** — упорядоченный набор структурированной информации, который хранится в электронном виде.  
**СУБД (DBMS)** — система управления базами данных: программа, с помощью которой создают БД/таблицы, добавляют/удаляют/изменяют данные, выполняют запросы и управляют доступом.

Что умеет СУБД (высокоуровнево):
- создавать базы и таблицы
- вставлять/обновлять/удалять данные
- извлекать данные запросами (в т.ч. по условиям)
- обеспечивать безопасность/права доступа

Примеры популярных СУБД: Oracle Database, Microsoft SQL Server, MySQL, PostgreSQL.

---

## 2) Реляционные базы: структура и термины
**Реляционная БД** — данные хранятся в таблицах, а таблицы связаны между собой отношениями (relations).

Ключевые элементы:
- **Сервер**: на одном сервере может быть несколько баз данных.
- **База данных (schema)**: контейнер таблиц.
- **Таблица (table)**: основной объект хранения данных.
- **Столбцы / поля (columns/fields)**: «что храним» (тип данных, имя столбца).
- **Строки / записи (rows/records)**: «конкретные записи» (набор значений по столбцам).
- **Ячейка**: пересечение строки и столбца.
- **Сущность (entity)**: объект предметной области (пользователь, заказ, товар), который описывается набором признаков → обычно хранится в таблице.
- **Запрос (query)**: команда на SQL для получения или изменения данных/структуры.
- **Выборка (result set)**: результат SELECT-запроса (набор данных), обычно существует в оперативной памяти.

---

## 3) SQL: что важно понимать
**SQL (Structured Query Language)** — язык запросов для работы с реляционными БД.

Главные идеи:
- SQL — **декларативный**: описываем *что хотим получить*, а не *как* это вычислять.
- Есть стандарт SQL, но у каждой СУБД есть **диалект** (доп. функции/синтаксис).

Операции над столбцами (часто используемые идеи):
- математика (сумма, разница, проценты)
- операции со строками
- вычисляемые (кастомные) колонки
- фильтрация по значениям столбцов (это будет позже)

```sql

```

---

## 5) Первый SQL: SELECT и FROM

### 5.1. Зачем нужны
- **SELECT** — что извлекаем (какие столбцы/значения)
- **FROM** — откуда извлекаем (из какой таблицы)

Базовый шаблон:
```sql
SELECT column1, column2
FROM table_name;
```

### 5.2. Вывести константу (SELECT без FROM)
Если в SELECT указать строку/число, то SQL просто выведет значение:
```sql
SELECT 'Hello world!';
SELECT 123;
```

### 5.3. Выбрать все столбцы
```sql
SELECT *
FROM employees;
```

### 5.4. Выбрать конкретные столбцы
```sql
SELECT first_name, last_name
FROM employees;
```

## 6) Комментарии в SQL
Однострочный комментарий:
```sql
-- это комментарий
SELECT 'Good morning';
```

Многострочный комментарий:
```sql
/* комментарий
   на несколько строк */
SELECT 1;
```

## 7) Стиль запросов (чтобы было читаемо)
Правила хорошего стиля:
1) Ключевые слова — **В ВЕРХНЕМ РЕГИСТРЕ**, имена таблиц/столбцов — в нижнем.
2) Разделяй ключевые слова и имена (пробелами, переносами строк).
3) Переноси список столбцов на отдельные строки в больших запросах.
4) Любой запрос заканчивай `;`

Пример форматирования:
```sql
SELECT
    first_name,
    last_name
FROM employees;
```

## 8) Подключение к серверу (MySQL Workbench — логика шагов)
Для подключения обычно нужны:
- hostname (сервер)
- username
- password

Общий процесс:
1) MySQL Workbench → вкладка **Home**
2) Если соединение уже есть — выбрать его
3) Если подключаешься впервые — нажать **+** и заполнить параметры
4) После подключения справа открыть **Schemas**, найти нужную БД (например, `northwind`)
5) Раскрыть базу → **Tables** → выбрать таблицу и смотреть столбцы/данные

## 9) Практика на `northwind` (минимальный набор)
```sql
-- 1) вывести строку
SELECT 'Hello world!';

-- 2) выбрать базу
USE northwind;

-- 3) вывести всю таблицу orders
SELECT *
FROM orders;

-- 4) вывести ship_name и ship_city
SELECT ship_name, ship_city
FROM orders;
```
